Dental Chair Display 2.0.1 Startup Fix

Root cause addressed:
- Some Android TV boxes return no WifiManager or reject multicast locks.
- The previous application initialized discovery before rendering the UI.
- A networking exception could terminate MainActivity immediately after a black launch screen.

Fixes:
- UI renders before any network operation.
- WifiManager is nullable and guarded.
- Multicast lock failures are non-fatal.
- Discovery and subnet scanning run safely in background.
- Subnet scan is batched instead of launching 254 simultaneous calls.
- Network errors switch to offline display mode instead of closing the app.
- All previous Display 2.0 features remain present.
